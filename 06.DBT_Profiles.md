# DBT Profiles YAML

The `profiles.yml` file is used by DBT to store connection information for your data warehouse. It typically resides in the `~/.dbt/` directory by default.

## Structure

A typical `profiles.yml` looks like:

```yaml
my_profile:
  target: dev
  outputs:
    dev:
      type: postgres
      host: localhost
      user: my_user
      password: my_password
      dbname: my_db
      schema: public
```

- **my_profile**: The profile name or dbt project name
- **target**: The default target (e.g., `dev`, `prod`).
- **outputs**: Connection details for each target.

## Selecting a Target Profile

You can override the default target using the `--target` flag:

```sh
dbt run --target prod
```

This will use the `prod` target configuration from your profile.

## Specifying Profile Directory Path

By default, DBT looks for `profiles.yml` in `~/.dbt/`. To use a different directory, use the `--profiles-dir` flag:

```sh
dbt run --profiles-dir /path/to/profiles/
```

This tells DBT to look for `profiles.yml` in the specified directory.

## Summary

- Use `--target` to select a target profile.
- Use `--profiles-dir` to specify a custom profiles directory.
- The `profiles.yml` file manages your DBT connection settings.
