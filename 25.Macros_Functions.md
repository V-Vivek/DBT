## Function Type Macros in dbt

In dbt, **function type macros** are reusable code snippets written in Jinja that return a value. They work like functions in programming languages and are used to encapsulate logic that can be reused across models, tests, or other macros.

### Syntax

```jinja
{% macro macro_name(arg1, arg2) %}
    {# logic here #}
    {{ return(value) }}
{% endmacro %}
```

### Example: Macro to Prefix Table Names

Suppose you want to prefix all table names with a specific environment name:

```jinja
{% macro prefixed_table(env, table_name) %}
    {{ return(env ~ '_' ~ table_name) }}
{% endmacro %}
```

**Usage in a model:**

```sql
select *
from {{ prefixed_table('prod', 'users') }}
```

This will render as:

```sql
select *
from prod_users
```

### Key Points

- They help avoid repetition and centralize logic.
- Can be used in models, seeds, tests, and other macros.
