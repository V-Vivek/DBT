# Jinja: Comments, Statements, and Expressions

Jinja is a templating engine commonly used in Python projects. It provides a way to embed dynamic content and logic in text files, such as HTML or SQL. Hereâ€™s how comments, statements, and expressions work in Jinja:

---

## 1. Comments

**Syntax:**  
```jinja
{# This is a comment #}
{# 
    This is a 
    multi-line comment 
#}

-- This is a SQL comment
SELECT * FROM users;
```

**Usage:**  
Jinja comments are ignored during rendering and they will not appera in the compiled SQL output. But the SQL comments will be included in the compiled SQL output.

**Output:**
```sql
-- This is a SQL comment
SELECT * FROM users;
```

---

## 2. Statements

**Syntax:**  
```jinja
{% statement %} ... {% endstatement %}
```

Statements control logic and flow, such as loops and conditionals.

**Examples (dbt context):**

- **Set Statement:**
    ```jinja
    {# This is an inline set statement #}
    {% set my_variable = 'Hello, World!' %}

    {# This is a multi-line set statement and hence requires endset #}
    {% set my_query %}
      SELECT *
      FROM users
      WHERE active = true
    {% endset %}
    ```


- **For Loop:**
    ```jinja
    {% for column in ['id', 'name', 'email'] %}
      {{ column }} {% if not loop.last %}, {% endif %} {# if condition is used to add a comma except for the last item #}
    {% endfor %}
    ```
    *Renders as:*
    ```
    id, name, email
    ```

> Note: Use - to remove whitespace or line breaks in Jinja statements  
> e.g., `{%- for item in items -%}`.
---

## 3. Expressions

**Syntax:**  
```jinja
{{ expression }}
```

Expressions output the result of Python expressions or variables.

**Examples:**
```jinja
{{ my_variable }}
{{ my_query }}
{{ 1 + 2 }}
{{ 'Hello' + ' ' + 'World' }}
```
**Output:**
```sql
Hello, World!
SELECT * FROM users WHERE active = true
3
Hello World
```

## Summary Table

| Feature     | Syntax                | Purpose                |
|-------------|-----------------------|------------------------|
| Comment     | `{# ... #}`           | Add notes, ignored     |
| Statement   | `{% ... %}`           | Logic, control flow    |
| Expression  | `{{ ... }}`           | Output values/results  |

---

Use these constructs to build dynamic, readable, and maintainable templates with Jinja.
